从动画来谈向上导航和向后导航的区别，顺带复习一遍启动模式

发表于 2016-11-14 | 分类于 技术 | | 你是第 个瞅这篇文章的哟

今天权威指南的时候学到了层级式导航的向上键（软件左上角的向左箭头）和临时性导航的后退键（即返回键）两个的实现机制和效果都是不同的，导致了动画效果也不相同。


导航键

假设有 A，B，C 三个 Activity，只能由 A 启动 B，再由 B 启动 C。这样它们三个在回退栈里的顺序自上往下就是 C——B——A 的顺序。

该文的前提是 Activity 继承的是 AppCompatActivity，并且都应用了 AppCompat 的工具栏主题

返回键

返回键就比较简单了，直接就是回到回退栈中的上一个 Activity，即在 C 中就是回到 B，在 B 中就是回到 A，没什么好说的。

有意思的是，我在手机上模拟的时候，操作过程中产生的动画可以直接与两种方式的实现机制联系起来，感觉特别有意思，也特别生动，所以水文一篇。

启动模式

Android 中的四种启动模式: standard, singleTop, singleTask, singleInstance。都是要在 AndoridManifest.xml 文件中通过<activity>的android:launchMode属性来指定。

  • standard: 是默认的启动模式，每当启动一个新的 Activity，它就会在回退栈中入栈，处于栈顶位置，不管该活动在返回栈中是否存在。
  • singleTop: 顾名思义，启动一个 Activity，如果该 Activity 已经在回退栈顶了，就不会启动一个新 Activity，而是使用已经在栈顶的 Activity。
  • singleTask: 根据名字也很好理解，启动的 Activity 若是在回退栈中已经存在了，启动的时候就会将回退栈中位于该 Acitivty 上面的 Acitivty 全部弹出回退栈，让启动的 Activity 活动处于栈顶。
  • singleInstance: 这个模式会启动一个新的回退栈来管理启动的 Activity。在不同的 APP 间要共享一个 Activity 时比较有用。
