用免费的阿里云给 VPS 做代理中转，爽的飞起

发表于 2015-12-03 | 分类于 技术 | | 你是第 个瞅这篇文章的哟

上周在 Telegram 群里水的时候，知道了阿里百川免费计划，可以给注册用户免费到 2016 年底，于是经过多方咨询（其实就是在群里水了半天），总算弄懂了如何用阿里云当跳板，代理身在美帝的 VPS。经实测，该方法可以基本实现 0 丢包，Ping 值 170 以下，简直是福音呐。特别是对那种 VPS 传输距离远的，丢包率高的，Ping 值高的，简直就是神器了。得赶紧趁着免费时间把我那些 Ping 值丢包爆表的 VPS 的余额完全消灭掉！

现在就把这个方法分享给大家咯。

首先当然是注册阿里百川计划，其实用淘宝帐号密码貌似就可以直接登录了地说。

首先登录，点击右上角“控制台”，进入控制台页面，然后进入“云服务控制台”，如图所示。

然后点击“镜像仓库-创建镜像-编辑镜像-构建并提交”，构建一个自己的 CentOS 镜像，名称什么的随意，填完后不用做任何修改，直接构建并提交，几秒钟后就构建完毕了。


此时回到云服务控制台，进入服务管理，创建服务，选择“自定义镜像”，选择刚才构建的镜像。


然后是参数设置，如图框出来的地方是需要填写或修改的，关键是协议要选“TCP”协议，其余随意。端口设置随你们，反正我是这样设置的。

然后就是两个“下一步”，然后“创建”了。

创建完毕后会回到“服务管理”界面。

首先启动服务，然后点击“WebSSH”，以 root 用户身份登入终端，安装“haproxy”，设置转发。
